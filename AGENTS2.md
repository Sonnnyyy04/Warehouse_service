## Что уже сделано

- Backend переведен на транзакционное создание товара:
  - `product + marker + initial batch` создаются атомарно.
- На backend добавлены серверные инварианты занятости:
  - нельзя создать/переместить короб или партию в занятую ячейку.
- История для админа теперь получает пользователей ролей `admin` и `worker`.
- Auth middleware исправлен:
  - HTML определяется через `Accept` корректнее;

- На клиенте экран `Работники` расширен до `Пользователи`:
  - можно создавать `worker` и `admin`;
  - добавлен выбор роли.
- `api_exception.dart` переписан в нормальный UTF-8.

### 6. Подумать над более безопасным flow для label/print авторизации

Сейчас `access_token` в query оставлен только для:

- `/api/v1/labels/qr`
- `/labels/print`
- `/labels/pdf`

Это лучше, чем было, но не идеал.

Позже можно улучшить:

Для диплома текущий вариант допустим, но это еще не финальная security-модель.
